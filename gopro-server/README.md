# gopro-webcam
# Yacht Monitoring from Tug (GoPro Lightweight Streamer)

低スペックなPC（Intel N200など）でも安定して動作するように設計された、超軽量・低遅延な簡易ビデオストリーミングサーバーです。
GoPro Webcamユーティリティなどの「仮想カメラ」を使用した際のフリーズ対策や、CPU負荷を最小限に抑えるための最適化が施されています。

主に、ローカルネットワーク経由で離れた場所からカメラの映像（ヨットの監視など）をWebブラウザでリアルタイムに確認する用途を想定しています。

## 🌟 主な特徴 (Features)

* **超軽量・低CPU負荷**: OpenCVの `grab()` と `retrieve()` を分離し、不要な画像変換処理をスキップすることでCPUリソースを節約します。
* **低遅延（ゼロラグ）**: 古いバッファを強制的に破棄し、常に最新のフレームのみを処理・配信します。
* **仮想カメラのフリーズ対策**: Windows環境において、仮想カメラ特有のデッドロックを防ぐため `DirectShow (cv2.CAP_DSHOW)` を強制指定しています。
* **簡単視聴**: 専用アプリは不要。同じネットワークに繋がっているスマホやPCのブラウザからアクセスするだけで映像を確認できます。

## 動作環境 (Prerequisites)

* **OS**: Windows 10 / 11 (DirectShowを使用するため)
* **カメラ**: GoPro HERO12 (GoPro Webcamユーティリティ経由) または一般的なWebカメラ
* **パッケージ管理**: [Rye](https://rye-up.com/)

## 🚀 インストール (Installation)

1. リポジトリをクローンします。

2. Ryeを使用して依存関係をインストールします。
Bash
```
rye sync
```

## 💻 使い方 (Usage)
起動方法
### パターンA: バッチファイルによるワンクリック起動（推奨）
プロジェクト内に作成された起動用の start.bat ファイルをダブルクリックするだけで、自動的にサーバーが立ち上がります。

### パターンB: コマンドラインからの起動

Bash
```
rye run python ./src/gopro_webcam/gp_streamer.py
```
視聴方法
サーバーを起動後、同じネットワークに接続されたデバイスのWebブラウザから以下のURLにアクセスします。

サーバーを実行しているPC自身で見る場合: http://localhost:5000 または http://127.0.0.1:5000

別のスマホやPCから見る場合: http://100.65.99.110:5000

注意: 起動時にブラウザ画面が真っ暗なまま映像が出ない場合は、GoPro Webcamユーティリティがフリーズしている可能性があります。PCのタスクトレイからGoPro Webcamを再起動してからお試しください。

⚙️ 設定のカスタマイズ (Configuration)
スクリプト（gp_streamer.py）上部の変数を変更することで、環境に合わせた調整が可能です。

Python
CAMERA_ID = 2        # カメラが認識されない場合は 0 や 1 に変更してください
TARGET_WIDTH = 1280  # 解像度の幅
TARGET_HEIGHT = 720  # 解像度の高さ
TARGET_FPS = 15      # 目標フレームレート（下げるほどCPU・ネットワーク負荷が下がります）
JPEG_QUALITY = 50    # 画質（0〜100。下げるほどデータ量が軽くなります）
